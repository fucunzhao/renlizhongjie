# 劳务中介智能客户服务系统 MVP

这是一个可部署的 MVP 版本，包含前端、后台接口和 SQLite 数据库。

## 当前功能

- 企业用工需求管理
- 全年用工日历排期
- 求职者数据库维护
- 岗位与求职者匹配推荐
- AI 本地知识库助手原型
- 后台数据库持久化保存
- 求职者自助登记页面：`/applicant.html`
- 私有知识库第一阶段：企业岗位规则、求职者画像自动沉淀为知识条目
- 私有知识库第二阶段：基于规则生成重点缺口、周结岗位、不用体检岗位、夜班人选等洞察

## 文件说明

- `index.html`：系统入口页面
- `styles.css`：页面样式
- `app.js`：前端交互逻辑
- `server.py`：后台服务、接口和数据库初始化
- `labor_service.db`：运行后自动生成的 SQLite 数据库

## 本地启动

在当前目录执行：

```bash
python server.py
```

然后浏览器打开：

```text
http://服务器IP:8000
```

如果是在本机测试，打开：

```text
http://127.0.0.1:8000
```

## 云服务器部署

1. 把整个文件夹上传到服务器。
2. 确认服务器安装了 Python 3。
3. 在服务器安全组或防火墙开放 `8000` 端口。
4. 进入项目目录运行：

```bash
python3 server.py
```

5. 访问：

```text
http://你的服务器IP:8000
```

## 数据库

系统首次启动会自动创建 `labor_service.db`，并写入乐颜提供的柳东、花岭、官塘等企业用工信息。

如果部署平台支持持久化目录，建议设置环境变量：

```text
DATA_DIR=/data
```

并把平台的持久化存储挂载到 `/data`。这样重新部署后，数据库不会丢失。

当前数据库表：

- `demands`：企业用工需求
- `workers`：求职者档案
- `chat_messages`：AI 助手对话记录
- `knowledge_entries`：企业岗位规则、求职者画像、业务维护沉淀出的私有知识条目

## 真实大模型接入建议

当前 AI 助手是规则型原型，已经能基于本地数据库回答岗位、缺口、推荐和文案问题。

正式接入大模型时，建议在 `server.py` 的 `/api/chat` 接口里增加：

- 本地数据库检索
- 企业需求和求职者资料摘要
- 大模型调用
- 敏感字段脱敏
- 回答结果留痕
